WHAT THIS DOES:

- Adds visibility, stock_deducted_at, out_of_stock_since to products
- Adds delivery_confirmed_at, delivered_by_role to orders
- Adds admin_approved, admin_approved_at to products
- Creates triggers to:
  * Auto-mark products as 'sold' when stock reaches 0
  * Prevent actions on cancelled orders
  * Finalize cancellations
  * Deduct stock ONLY on delivery confirmation
  * Restore stock when order is cancelled
  * Prevent actions on cancelled orders

------- FEATURE 6: ADMIN PRODUCT APPROVAL NOTIFICATIONS -------
FILE: lib/pages/add_edit_product_page.dart

ADD THIS METHOD to handle admin notifications:
    
    Future<void> _notifyAdminsOfNewListing(int productId) async {
      try {
        final user = supabase.auth.currentUser;
        if (user == null) return;

        final product = await supabase
            .from('products')
            .select()
            .eq('id', productId)
            .maybeSingle();

        if (product == null) return;

        final admins = await supabase
            .from('profiles')
            .select('id')
            .eq('role', 'admin');

        for (final admin in admins) {
          await supabase
              .from('notifications')
              .insert({
                'user_id': admin['id'],
                'sender_id': user.id,
                'title': 'New Product Listing for Review',
                'body': '${product['title']} by ${user.email} is pending approval.',
                'type': 'product_approval',
                'product_id': productId,
                'data': {
                  'product_id': productId,
                  'action_type': 'approve'
                },
              });
        }
      } catch (e) {
        print('Error notifying admins: $e');
      }
    }

LOCATE: Where products are created successfully
ADD THIS LINE after successful product creation:

    Future<void> _handleProductApproval(
      NotificationModel notification, 
      bool approved
    ) async {
      final productId = notification.data['product_id'];
      if (productId == null) return;

      try {
        final user = supabase.auth.currentUser;
        if (user == null) return;

        await supabase
            .from('products')
            .update({
              'admin_approved': approved,
              'admin_approved_at': DateTime.now().toIso8601String(),
              'validated': approved,
              'status': approved ? 'active' : 'hidden',
            })
            .eq('id', productId);

        await supabase
            .from('notifications')
            .update({'is_read': true})
            .eq('id', notification.id);

        if (!mounted) return;
        ScaffoldMessenger.of(context).showSnackBar(
          SnackBar(
            content: Text(approved ? 'Product approved' : 'Product revoked'),
          ),
        );

        await _loadNotifications();
      } catch (e) {
        if (!mounted) return;
        ScaffoldMessenger.of(context).showSnackBar(
          SnackBar(content: Text('Error: $e')),
        );

    }

ADD THIS METHOD to handle cancellation:
    
    Future<void> _cancelOrder(int orderId) async {
      try {
        final user = supabase.auth.currentUser;
        if (user == null) return;

        final confirm = await showDialog<bool>(
          context: context,
          builder: (ctx) => AlertDialog(
            title: Text('Cancel Order?'),
            content: Text(
              'Are you sure? This action is FINAL. '
              'You won\'t be able to perform any actions afterward.'
            ),
            actions: [
              TextButton(
                onPressed: () => Navigator.pop(ctx, false),
                child: Text('Keep Order'),
              ),
              TextButton(
                onPressed: () => Navigator.pop(ctx, true),
                child: Text('Cancel Order', style: TextStyle(color: Colors.red)),
              ),
            ],
          ),
        );

        if (confirm != true) return;

        await supabase
            .from('orders')
            .update({
              'cancelled_by_role': 'buyer',
              'status': 'cancelled',
            })
            .eq('id', orderId)
            .eq('buyer_id', user.id);

        if (!mounted) return;
        ScaffoldMessenger.of(context).showSnackBar(
          SnackBar(
            content: Text('Order cancelled. No further actions possible.'),
          ),
        );

        await _loadOrders();
      } catch (e) {
        if (!mounted) return;
        ScaffoldMessenger.of(context).showSnackBar(
          SnackBar(content: Text('Error: $e')),
        );
      }
    }

===============================================================
PART 3: HOW THE FEATURES WORK TOGETHER
===============================================================

SHIPPING:
1. Buyer adds products to cart
2. Cart displays "Shipping: Free" (always 0)
3. Buyer proceeds to checkout
4. Checkout shows shipping as 0
5. Order created with shipping_price = 0

PRODUCT VISIBILITY:
1. Seller creates product → Admin gets notification
2. Admin can Approve or Revoke
3. When approved: status = 'active', visibility = 'visible'
4. When revoked: status = 'hidden', visibility = 'hidden'
5. When stock reaches 0: visibility = 'sold' (visible with SOLD label)
6. Product appears on homepage with "SOLD OUT" label

STOCK MANAGEMENT:
1. Order created → Stock NOT deducted (reserved)
2. Seller confirms shipment → Still not deducted
3. Seller confirms delivery → Stock IS deducted
4. If order cancelled:
   - All actions disappear
   - Stock is restored
   - Order shows "CANCELLED - No actions available"

CANCELLATION:
1. Buyer/Seller can cancel order (one-time action)
2. cancelled_by_role is set to 'buyer' or 'seller'
3. cancellation_finalized = TRUE (prevents further actions)
4. All action buttons hidden
5. Order grayed out with cancellation message
6. Stock restored to product

ADMIN ACTIONS:
1. New product listed → Admin notified
2. Admin views notification
3. Admin can: APPROVE (make visible) or REVOKE (hide)
4. Decision is saved immediately
